Making a Phishing-Prevention JavaScript Library


Let's make the JavaScript Library for phishing prevention.

Here are the steps to make our website secure against phishing attacks.
1. Get all anchor tags in the webpage. We can do this by using the document.getElementsByTagName() function.
2. Add an on click listener to each anchor tag and supress its default action, so we won't be redirected anymore when an anchor tag is clicked.

Combining steps 1 and 2, the code will be:

let anchorTags = document.getElementsByTagName('a')
for (let i = 0; i < anchorTags.length; i++){
  anchorTags[i].addEventListener('click', (e) => {
    e.preventDefault();
  })
}


3. We need to get the hostname of in the anchor tag. This can be done using anchorTags[i].hostname
The .hostname property automatically converts the special unicode characters such as https://ⓖⓞⓞⓖⓛⓔ.ⓒⓞⓜ but if we want to normalize this URL in other ways, we can use the String.prototype.normalize. The code for this is:
let str = 'https://ⓖⓞⓞⓖⓛⓔ.ⓒⓞⓜ'
str.normalize('NFKD') // you may use NFKD or NFKC


The image below shows the function of NFKD and NFKC. If this way of normalizing URLs will be done, take note that it will not convert the dot (｡) in https://google｡com to https://google.com. This means that we still need to add another step to replace this dot (｡) to a normal dot (.)


4. We can now check the host name if it is included in the whitelist or blacklist.

5. We need to add an additional step, because the hostname property from step 3 only simplifies the IPv6/IPv4 Address Embedding (http://[::ffff:142.250.199.78]) to [::ffff:8efa:c74e] and not to 142.250.199.78. To convert this to the proper IP address, the steps that we will be doing are:
1) If the hostname matches that format of [::ffff:XXXX:XXXX] (where XXXX is a hexadecimal character). This can be done using the right regex match.
if(hostname.match(/^\[::ffff:[a-f0-9]{4}:[a-f0-9]{4}\]/)) {
  console.log(hostname)
}

2) Convert the XXXX:XXXX part to the integer notation of the IP address.
if(hostname.match(/^\[::ffff:[a-f0-9]{4}:[a-f0-9]{4}\]/)) {
  hostname = hostname.substring(1, hostname.length - 1) // remove the opening and closing brackets in the IP
  let parts = hostname.split(':') // split the hostname by the colon character
  hostname = parts[3]+parts[4]  // join the last 2 parts
  hostname = parseInt(hostname, 16) // convert the hex to integer
  console.log(hostname)
}

3) Convert the integer notation to a proper IP address (in the form of XXX.XXX.XXX.XXX). Let's make a function for this.
const intToIP = (int) => {
    let part1 = int & 255;
    let part2 = ((int >> 8) & 255);
    let part3 = ((int >> 16) & 255);
    let part4 = ((int >> 24) & 255);

    return part4 + "." + part3 + "." + part2 + "." + part1;
}

4) Now that the IP is in proper notation, we can check it again for whitelist and blacklist.

6. Display the appropriate response. A “danger” modal if the link is blacklisted. A “warning” modal if the link is normal. And none if the link is whitelisted.
